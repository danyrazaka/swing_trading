Assistant d'Aide à la Décision pour le Swing Trading

Ce projet n'est pas un bot de trading automatique. C'est un assistant personnel conçu pour vous aider à prendre des décisions de swing trading manuel. Il identifie les opportunités et vous donne des signaux, mais la décision finale et l'exécution de l'ordre vous appartiennent.

Installation et Mise en Place

Étape 0 : Prérequis

Assurez-vous que Python est installé sur votre machine (version 3.8 ou supérieure). Vous pouvez le télécharger depuis python.org. Lors de l'installation, cochez la case "Add Python to PATH".

Étape 1 : Obtenir les fichiers du projet

Créez un dossier principal pour le projet (par exemple, swing-trader-ia) et placez-y tous les fichiers fournis : main.py, requirements.txt, et le dossier src avec tout son contenu.

Étape 2 : Installer les dépendances

Ouvrez votre terminal (ou 'Invite de commandes' sur Windows) et naviguez jusqu'au dossier du projet.

cd chemin/vers/votre/dossier/swing-trader-ia

Il est fortement recommandé d'utiliser un environnement virtuel.

python -m venv venv
# Sur Windows: venv\Scripts\activate | Sur macOS/Linux: source venv/bin/activate

Installez toutes les bibliothèques nécessaires en une seule commande.

pip install -r requirements.txt --upgrade

Le programme est maintenant prêt à être utilisé !

Le Workflow d'Utilisation Recommandé

Routine 1 : Le Scan du Matin (Début de journée)

python main.py scan

Trouve le Top 5 des opportunités du jour et les sauvegarde dans daily_candidates.txt.

Routine 2 : L'Entraînement Ciblé (Juste après le scan)

python main.py train

Entraîne les modèles IA uniquement pour les 5 candidats du jour. C'est rapide et efficace.

Routine 3 : Le Conseil en Cours de Journée (À tout moment)

python main.py advise

Donne un conseil pour chaque candidat du jour, en utilisant les modèles fraîchement entraînés.

Tâche Optionnelle : L'Entraînement de Fond (Hebdomadaire)

De temps en temps (par exemple, le week-end), vous pouvez lancer un entraînement complet pour vous assurer que tous les modèles, même ceux qui ne sont pas dans le top 5, sont à jour.

python main.py train --all

Personnalisation : Devenez le Chef d'Orchestre

L'intelligence du programme dépend des actifs que vous lui demandez de surveiller. Vous avez le contrôle total sur cette liste (la "watchlist").

  1. Ouvrez le fichier src/investment_selector.py.
  2. Localisez la liste XTB_ASSETS.
  3. Modifiez cette liste en ajoutant ou en supprimant des tickers pour qu'elle corresponde aux marchés qui vous intéressent.

Astuce : Pour trouver le bon ticker pour un actif, cherchez sur Google "Nom de l'action Yahoo Finance".

Plongée au Cœur du Projet : L'Apprentissage par Renforcement (RL)

Comment l'IA apprend-elle à trader ? Elle utilise une technique appelée Apprentissage par Renforcement. Le meilleur moyen de le comprendre est d'imaginer que l'on entraîne une IA à jouer à un jeu vidéo dont le but est de maximiser le score (la valeur du portefeuille). L'IA joue à ce jeu des dizaines de milliers de fois sur les données historiques.

Architecture Détaillée de l'IA (PPO)

Le "cerveau" de notre IA est un modèle PPO (Proximal Policy Optimization) avec une politique "MlpPolicy". Cela signifie qu'il est composé d'un réseau de neurones (Multi-Layer Perceptron). Ce réseau a une architecture "Acteur-Critique" : il a deux têtes de sortie qui travaillent en tandem.

1. Entrée (Observation)

État actuel du marché et du portefeuille.

→
2. Cerveau de l'IA (Réseau de Neurones)

Couches de Neurones Cachées
(Analyse des patterns)

Tête 1 : L'Acteur (Politique)
Décide : "Quelle action faire ?"

Tête 2 : Le Critique (Valeur)
Estime : "Cette situation est-elle bonne ?"

→
3. Sortie (Décision)

L'action la plus probable (Acheter, Vendre ou Conserver).

  • L'Acteur est la partie qui choisit l'action concrète (ACHETER, VENDRE, CONSERVER).
  • Le Critique évalue la situation actuelle. Il aide l'Acteur en lui disant si les choix qu'il fait dans une situation donnée mènent à de bonnes ou de mauvaises récompenses sur le long terme.

Pendant l'entraînement, l'algorithme PPO utilise les estimations du Critique pour guider l'Acteur à prendre de meilleures décisions, de manière stable et progressive.

Exemple Concret : Une Décision en une Fraction de Seconde

Imaginons que nous sommes le matin et que le `scan` a identifié NVDA comme une opportunité. Vous lancez `advise`.

La Situation

Le programme récupère les dernières données pour NVDA et voit que le prix est en hausse, le volume est important, le RSI est à 65 (fort mais pas encore sur-acheté). Votre portefeuille pour cet actif est vide (solde de 10 000€, 0 action détenue).

L'Observation (Entrée)

Toutes ces informations sont converties en un simple tableau de nombres (un `array` Numpy) qui est présenté au "cerveau" :

# Ce qui est envoyé au modèle model.predict()
[
  130.5,    // Prix de Clôture
  125.2,    // Moyenne Mobile 10j
  118.1,    // Moyenne Mobile 30j
  110.4,    // Moyenne Mobile 50j
  65.3,     // RSI 14j
  10000.0,  // Solde du portefeuille
  0.0       // Actions détenues
]

La Prédiction (Sortie)

Le réseau de neurones analyse ce tableau en une fraction de seconde. Sa tête "Acteur", conditionnée par des milliers d'entraînements, détermine que dans une telle situation, l'action qui a le plus souvent mené à une récompense positive est "ACHETER". Il retourne un simple chiffre :

2

Le programme interprète ce chiffre (0=Vendre, 1=Conserver, 2=Acheter) et vous affiche le conseil final : Conseil pour NVDA : **ACHETER**.
